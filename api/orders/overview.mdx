## Overview

The Orders API allows you to create, retrieve, update, and delete customer orders. Each order contains customer information, order details, and a collection of order line items (Detalles).

## Authentication

All order endpoints require authentication. Include the JWT token in the Authorization header:

```
Authorization: Bearer {token}
```

## Entity Structure

The `Pedido` (Order) entity has the following structure:

```csharp
public class Pedido
{
    public int idPedido { get; set; }
    public DateTime fecha { get; set; } = DateTime.Now;
    public decimal total { get; set; }
    public string metodoPago { get; set; } // "tarjeta" o "efectivo"
    public int idUsuario { get; set; }
    public virtual Usuario Usuario { get; set; }
    public virtual ICollection<Detalle> detalles { get; set; }
}
```

### Order Details (Detalles)

Each order contains a collection of line items:

```csharp
public class Detalle
{
    public int idDetalle { get; set; }
    public int cantidad { get; set; }
    public decimal precioUnitario { get; set; }
    public int idProducto { get; set; }
    public virtual Producto producto { get; set; }
}
```

## POST /api/Pedidos

Create a new order.

### Request Body

<ParamField body="fecha" type="datetime">
  Order date (defaults to current date/time if not provided)
</ParamField>

<ParamField body="total" type="decimal" required>
  Total order amount (decimal with 2 precision)
</ParamField>

<ParamField body="metodoPago" type="string" required>
  Payment method: "tarjeta" or "efectivo" (max 50 characters)
</ParamField>

<ParamField body="idUsuario" type="integer" required>
  Customer user ID (must be a valid existing user)
</ParamField>

<ParamField body="detalles" type="array" required>
  Array of order line items
  
  <ParamField body="cantidad" type="integer" required>
    Quantity of the product
  </ParamField>
  
  <ParamField body="precioUnitario" type="decimal" required>
    Unit price at the time of order
  </ParamField>
  
  <ParamField body="idProducto" type="integer" required>
    Product ID
  </ParamField>
</ParamField>

### Request Example

```json
{
  "fecha": "2026-03-04T14:30:00",
  "total": 91.98,
  "metodoPago": "tarjeta",
  "idUsuario": 5,
  "detalles": [
    {
      "cantidad": 2,
      "precioUnitario": 45.99,
      "idProducto": 1
    }
  ]
}
```

### Success Response (201 Created)

Returns the created order object with the assigned `idPedido` and a `Location` header pointing to the new resource.

### Error Response (400 Bad Request)

```json
"Error message describing what went wrong"
```

### Code Example

```bash
curl -X POST https://api.huellitas.com/api/Pedidos \
  -H "Authorization: Bearer {your_token_here}" \
  -H "Content-Type: application/json" \
  -d '{
    "total": 91.98,
    "metodoPago": "tarjeta",
    "idUsuario": 5,
    "detalles": [
      {
        "cantidad": 2,
        "precioUnitario": 45.99,
        "idProducto": 1
      }
    ]
  }'
```

```javascript

}
```

## Important Notes

<Warning>
- Deleting an order will also delete all associated order details (Detalles) due to cascade delete
- Ensure stock levels are adjusted appropriately when orders are created or deleted
- The total should match the sum of (cantidad × precioUnitario) for all line items
</Warning>

### Best Practices

1. **Calculate Total**: Always calculate the order total on the server side to prevent manipulation
2. **Stock Validation**: Verify product availability before creating orders
3. **Price Consistency**: Store the unit price in `Detalle` to maintain historical accuracy
4. **Order Status**: Consider adding an order status field for order lifecycle management
5. **Soft Delete**: Consider soft-delete for orders to maintain historical data

## Related Entities

- **Usuario** (`Huellitas.Core/Entities/Usuario.cs`): Customer who placed the order
- **Producto** (`Huellitas.Core/Entities/Producto.cs`): Products included in order details
- **Detalle** (`Huellitas.Core/Entities/Detalle.cs`): Individual line items in the order
